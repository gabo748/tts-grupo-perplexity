# Data Model: File-Based I/O

**Date**: 2025-11-24
**Feature**: [Text-to-Speech and Translation CLI](../spec.md)

## Overview

This feature does not use a traditional database. The data model consists of the files that the CLI tool consumes and produces.

## Entities

### 1. Input Text File

-   **Description**: A plain text file (`.txt`) provided by the user.
-   **Fields**:
    -   `content`: The raw text content of the file.
-   **Validation Rules**:
    -   The file must exist at the provided path.
    -   The file must be readable.
    -   The file should contain UTF-8 encoded text.

### 2. Input Audio File

-   **Description**: An audio file (e.g., `.mp3`, `.wav`) provided by the user for translation. This can be a file previously generated by the `tts` command or from another source.
-   **Validation Rules**:
    -   The file must exist at the provided path.
    -   The file must be in a format supported by the OpenAI Whisper API.

### 3. Output Audio File

-   **Description**: An audio file (e.g., `.mp3`) generated by the `tts` or `translate` commands.
-   **Attributes**:
    -   The file contains the synthesized speech or the translated audio.
    -   The file name can be specified by the user or generated automatically.
